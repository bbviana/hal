Menu

Menus allow you to take an action by selecting from a list revealed upon opening a temporary, new sheet of material.

A Menu is typically used in a Popover.

Examples

Simple examples
import React from 'react';
    import Paper from 'material-ui/Paper';
    import Menu from 'material-ui/Menu';
    import MenuItem from 'material-ui/MenuItem';

    const style = {
    display: 'inline-block',
    margin: '16px 32px 16px 0',
    };

    const MenuExampleSimple = () => (
    <div>
    <Paper style={style}>
    <Menu>
    <MenuItem primaryText="Maps" />
    <MenuItem primaryText="Books" />
    <MenuItem primaryText="Flights" />
    <MenuItem primaryText="Apps" />
    </Menu>
    </Paper>
    <Paper style={style}>
    <Menu>
    <MenuItem primaryText="Refresh" />
    <MenuItem primaryText="Help &amp; feedback" />
    <MenuItem primaryText="Settings" />
    <MenuItem primaryText="Sign out" />
    </Menu>
    </Paper>
    </div>
    );

    export default MenuExampleSimple;

Menu Properties

Name Type Default Description
autoWidth bool true If true, the width of the menu will be set automatically according to the widths of its children, using proper keyline increments (64px for desktop, 56px otherwise).
children node The content of the menu. This is usually used to pass MenuItem elements.
desktop bool false If true, the menu item will render with compact desktop styles.
disableAutoFocus bool false If true, the menu will not be auto-focused.
initiallyKeyboardFocused bool false If true, the menu will be keyboard-focused initially.
listStyle object Override the inline-styles of the underlying List element.
maxHeight number null The maximum height of the menu in pixels. If specified, the menu will be scrollable if it is taller than the provided height.
menuItemStyle object Override the inline-styles of menu items.
multiple bool false If true, value must be an array and the menu will support multiple selections.
onChange function () => {} Callback function fired when a menu item with value not equal to the current value of the menu is touch-tapped.

Signature:
function(event: object, value: any) => void
event: TouchTap event targeting the menu item.
value: If multiple is true, the menu's value array with either the menu item's value added (if it wasn't already selected) or omitted (if it was already selected). Otherwise, the value of the menu item.
onEscKeyDown function () => {} Callback function fired when the menu is focused and the Esc key is pressed.

Signature:
function(event: object) => void
event: keydown event targeting the menu.
onItemTouchTap function () => {} Callback function fired when a menu item is touch-tapped.

Signature:
function(event: object, menuItem: object, index: number) => void
event: TouchTap event targeting the menu item.
menuItem: The menu item.
index: The index of the menu item.
onMenuItemFocusChange function Callback function fired when the focus on a MenuItem is changed. There will be some "duplicate" changes reported if two different focusing event happen, for example if a MenuItem is focused via the keyboard and then it is clicked on.

Signature:
function(event: object, newFocusIndex: number) => void
event: The event that triggered the focus change. The event can be null since the focus can be changed for non-event reasons such as prop changes.
newFocusIndex: The index of the newly focused MenuItem or -1 if focus was lost.
selectedMenuItemStyle object Override the inline-styles of selected menu items.
style object Override the inline-styles of the root element.
value any If multiple is true, an array of the values of the selected menu items. Otherwise, the value of the selected menu item. If provided, the menu will be a controlled component. This component also supports valueLink.
valueLink object ValueLink for the menu's value.
width propTypes.stringOrNumber The width of the menu. If not specified, the menu's width will be set according to the widths of its children, using proper keyline increments (64px for desktop, 56px otherwise).

Other properties (not documented) are applied to the root element.

MenuItem Properties

Name Type Default Description
anchorOrigin propTypes.origin {horizontal: 'right', vertical: 'top'} Location of the anchor for the popover of nested MenuItem elements.
animation function Override the default animation component used.
checked bool false If true, a left check mark will be rendered.
children node Elements passed as children to the underlying ListItem.
disabled bool false If true, the menu item will be disabled.
focusState enum:
 'none'
 'focused'
 'keyboard-focused'
 'none' The focus state of the menu item. This prop is used to set the focus state of the underlying ListItem.
innerDivStyle object Override the inline-styles of the inner div.
insetChildren bool false If true, the children will be indented. This is only needed when there is no leftIcon.
leftIcon element The SvgIcon or FontIcon to be displayed on the left side.
menuItems node MenuItem elements to nest within the menu item.
onTouchTap function Callback function fired when the menu item is touch-tapped.

Signature:
function(event: object) => void
event: TouchTap event targeting the menu item.
primaryText node Can be used to render primary text within the menu item.
rightIcon element The SvgIcon or FontIcon to be displayed on the right side.
secondaryText node Can be used to render secondary text within the menu item.
style object Override the inline-styles of the root element.
value any The value of the menu item.

Other properties (not documented) are applied to the root element.
